PART 1

오픈소스와 Git의 기초 (1~6강)

오픈소스의 철학과 Git의 탄생 배경, 그리고 필수 환경 설정

오픈소스 소프트웨어 (OSS)

소스코드가 공개되어 누구나 확인, 수정, 재배포할 수 있는 소프트웨어입니다. 단순한 '무료'가 아닌 자유(Freedom)와 집단지성을 강조합니다.

Git과 GitHub

  • Git: 리누스 토발즈가 개발한 분산 버전 관리 시스템(DVCS). 모든 개발자가 전체 코드 사본을 가집니다.
  • GitHub: Git 저장소 호스팅 및 개발자들의 SNS 역할을 하는 웹 서비스입니다.
필수 초기 설정 (최초 1회)
git config --global user.name "이름" 사용자 이름 등록
git config --global user.email "메일" 사용자 이메일 등록
git config --global core.autocrlf true 줄바꿈 자동 변환 (윈도우 필수)
git config --global init.defaultBranch main 기본 브랜치명을 main으로 설정
PART 2

버전 관리의 3대 영역과 조작 (7~12강)

Git의 핵심인 Working Directory, Staging Area, Repository의 이해

3가지 핵심 영역 (The Three States)

Working Directory 작업 디렉토리

눈에 보이는 실제 작업 폴더
(버전 관리 전)

git add
Staging Area 스테이징 영역

커밋할 파일을 올려두는
대기 장소 (Index)

git commit
Repository Git 저장소

버전이 영구 저장되는
.git 폴더

📄 파일 상태 생명주기

  • Untracked Git이 추적하지 않는 새 파일
  • Modified 수정됐지만 스테이지에 없음
  • Staged 커밋 준비 완료된 상태

🔍 비교 (Diff) & 복구 (Restore)

git diff
작업 디렉토리 vs 스테이징 비교
git diff --staged
스테이징 vs 최신 커밋 비교
git restore [파일]
작업 내용 취소 (최근 스테이지 상태로)
git restore --staged [파일]
스테이징 내리기 (Unstage)
PART 3

브랜치 전략과 원격 협업 (13~18강)

독립적인 작업 공간 브랜치와 GitHub 연동

🏷 버전 태깅 (Tag)

SemVer (v1.0.0) 형식을 따릅니다.

git tag v1.0 (Lightweight)
git tag -a v1.0 -m "msg" (Annotated)

🌿 브랜치 (Branch)

원본을 건드리지 않고 독립적으로 작업하기 위한 복사본.

git switch -c [이름] (생성 및 이동)
HEAD: 현재 브랜치의 최신 커밋 포인터

원격 저장소 (Remote) 핵심

  • Clone: 원격 저장소 복제 (git clone)
  • Push: 로컬 커밋 업로드 (git push origin [브랜치])
  • Pull: 가져와서 자동 병합 (git pull)
  • Fetch: 가져오기만 하고 병합 안 함 (git fetch)
  • Tip: 2021년부터 패스워드 대신 PAT (토큰) 인증 필수
기말 핵심

고급 기능 및 라이선스 (19~30강)

라이선스, Stash, 병합 전략, Rebase, 그리고 Reset vs Revert

⚖️ 오픈소스 라이선스 심화

구분 Copyleft (강한 의무) Permissive (약한 의무)
특징 코드 수정 시 공개 의무 발생 공개 의무 거의 없음, 자유로운 사용
종류 GPL (2차 저작물도 공개), LGPL MIT, Apache (특허 조항 포함)

📦 임시 저장 (Stash)

작업 중 급하게 브랜치를 이동해야 할 때 사용

git stash
스택에 임시 저장 후 작업 트리 비움
git stash pop
불러오고 목록에서 삭제
git stash apply
불러오고 목록에 유지

🔀 병합 (Merge) 3가지 전략

  • Fast-forward: 일렬 상태, 포인터만 이동
  • 3-way Merge: 갈라진 브랜치, 새 커밋 생성
  • Squash Merge: 모든 커밋을 하나로 뭉쳐서 병합

이력 수정: Rebase & Amend

  • Rebase: 브랜치의 베이스를 변경하여 이력을 선형(한 줄)으로 만듦. (원격 커밋 금지)
  • Amend: git commit --amend로 최신 커밋 메시지나 내용 수정.

⚡️ [★중요] 되돌리기: Reset vs Revert

가장 시험에 자주 나오는 핵심 포인트입니다.

Reset (되돌리기)

"아예 없던 일로 만들기"

  • 과거 시점으로 이동, 이후 이력 삭제
  • 혼자 작업하는 로컬 저장소에서 사용
  • 옵션:
    • --hard (모두 삭제)
    • --mixed (변경내용 남김-기본값)
    • --soft (커밋만 취소, 스테이지 유지)
VS

Revert (취소)

"실수 정정 커밋 만들기"

  • 반대 작업을 수행하는 새 커밋 생성
  • 이력 보존됨 (협업 시 안전)
  • 이미 원격(Push)된 커밋을 취소할 때 필수
  • --no-edit: 메시지 자동 생성
